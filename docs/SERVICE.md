# 客服模块功能介绍

## 概述

客服模块为 WebChat 系统提供在线客服功能，支持用户咨询、客服接待、会话管理、绩效统计等完整流程。

## 用户端功能

### 1. 查看客服状态

- 显示在线客服数量
- 显示当前等待人数
- 显示可用客服列表及空余位置

### 2. 加入等待队列

- 点击"联系客服"进入等待队列
- 显示当前排队位置
- 显示预计等待时间

### 3. 实时会话

- 与客服进行实时文字聊天
- 支持发送图片
- 消息已读状态跟踪
- 打字状态指示器
- **页面刷新自动恢复会话**：重新加载最近消息

### 4. 结束会话

- 用户可主动结束会话
- **两端都会收到结束通知**：用户和客服都能实时收到会话结束事件
- 结束会话后弹出评价弹窗
- 评分 1-5 星，可填写评价备注

## 客服端功能 (工作台)

### 1. 会话管理

- **查看自己的会话**：只会显示当前客服接待的会话（通过 serviceId 筛选）
- 点击会话查看聊天记录
- 实时接收新消息
- 接待新用户（从队列分配）
- **会话结束实时通知**：收到 `service_session_ended` 事件自动刷新列表

### 2. 消息收发

- 发送文字消息
- 发送图片
- 使用快捷回复（预设常用语）
- 查看历史消息

### 3. 会话操作

- **转移会话**: 将当前会话转接给其他在线客服
- **结束会话**: 结束当前用户会话（用户也会收到通知）

### 4. 备注功能

- **添加备注**: 记录客户的重要信息
- **设置标签**: 标记客户类型（投诉/咨询/VIP/紧急等）

### 5. 状态管理

- 在线/忙碌/离线 三种状态
- 设为忙碌时不会接收新用户

## 管理端功能

### 1. 客服账号管理

- 创建新的客服账号
- 设置客服状态（在线/忙碌/离线）
- 查看客服当前接待数
- 重置客服密码

### 2. 数据统计

- **总会话数**: 历史总会话数量
- **进行中**: 当前活跃会话数
- **已完成**: 已结束的会话数
- **等待队列**: 当前等待人数
- **在线客服**: 当前在线客服数

### 3. 绩效统计

- 个人总会话数
- 已完成会话数
- 平均会话时长

### 4. 会话历史

- 查看所有历史会话记录（分页）
- 支持按状态筛选（全部/排队中/进行中/已结束）
- 显示：用户、客服、状态、开始/结束时间、会话时长
- **新增显示**：评分、评价备注、内部备注、标签

### 5. 快捷回复管理

- 添加/删除快捷回复
- 按客服ID分开管理

## 技术实现

### 队列机制

使用 Redis Sorted Set 实现等待队列：

```
Key: chat:service:queue:order
Value: ZSET [userId -> timestamp]

- 用户加入: ZADD (score = timestamp)
- 客服接待: ZPOPMIN
- 用户离开: ZREM
- 等待人数: ZCARD
```

### 会话状态

```
┌──────────┐    客服接待    ┌──────────┐   用户离开/超时   ┌──────────┐
│ waiting  │ ───────────▶ │ chatting │ ──────────────▶ │ finished│
└──────────┘               └──────────┘                   └──────────┘
       │                        │
       │                        │
   15分钟超时              30分钟无活动
       │                        │
       ▼                        ▼
   离开队列              自动结束
```

### 超时服务

`ServiceSessionTimeoutService` 定时检查超时会话：

- **每 60 秒执行一次**
- 排队等待超过 15 分钟 → 自动离开队列
- 聊天中 30 分钟无消息 → 自动结束会话

### 数据库索引

```java
@CompoundIndex(name = "service_status_idx", def = "{'serviceId': 1, 'status': 1}")
@CompoundIndex(name = "user_status_idx", def = "{'userId': 1, 'status': 1}")
```

### 性能优化

1. **Redis 缓存**: 服务状态缓存 30 秒
2. **虚拟列表**: 前端消息列表虚拟滚动
3. **请求缓存**: 前端 API 请求 5 秒缓存
4. **超时处理**: 定时任务检查超时会话

## 使用流程

### 用户使用流程

```
1. 进入"我的"页面
2. 点击"联系客服"
3. 系统显示客服状态
4. 点击"联系客服"加入队列
5. 等待分配（显示排队位置）
6. 分配成功后开始聊天
7. 刷新页面自动加载最近消息
8. 聊完后结束会话
9. 弹出评价弹窗
10. 提交评价
```

### 客服工作流程

```
1. 登录管理后台
2. 进入"客服工作台"
3. 设置状态为"在线"
4. 查看自己的会话列表（默认只显示自己的会话）
5. 点击"接待下一个"
6. 系统从队列分配用户
7. 与用户聊天
8. 可使用快捷回复
9. 可添加备注/标签
10. 用户结束或主动结束会话
11. 收到会话结束通知，列表自动刷新
12. 继续接待下一个
```

## 注意事项

1. 客服需要先在管理后台创建账号，并分配 service 角色
2. 客服状态为 offline 时不会被分配新用户
3. 客服达到最大接待数时不会收到新用户
4. 会话列表默认按创建时间倒序显示（最新在前）
5. 结束会话时，用户和客服两端都会收到 `service_session_ended` 事件通知
